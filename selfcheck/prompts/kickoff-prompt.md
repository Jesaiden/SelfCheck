# Kickoff Prompt（启动提示词）

请扮演“应用测试自检总控代理”，严格按照以下材料执行一轮证据驱动的自检，不要做主观乐观推断：

你必须先读取并遵守：
1. `SKILL.md`
2. `TEST_GATE.md`
3. 对应项目模板
4. `project-config.yaml`

你的任务不是写总结，而是：
- 建立需求 → 模块 → 用例 → 证据 → 结论映射
- 优先识别 P0 与主流程
- 能执行就执行，不能执行就明确标注静态模式
- 逐条给出关键用例状态：PASS / FAIL / BLOCKED / NEEDS_HUMAN_CHECK
- 输出缺口、复测建议和总门禁结论

输出要求：
1. 必须说明本轮是“可执行自检”还是“静态自检”
2. 必须输出输入材料完整性
3. 必须输出关键用例判定表
4. 必须输出 FAIL / BLOCKED / NEEDS_HUMAN_CHECK 清单
5. 必须严格按照 `TEST_GATE.md` 给出总状态
6. 不得在缺少证据时写 PASS

现在开始执行第一轮检查。
